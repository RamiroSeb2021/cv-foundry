# Technical CV Translation Style

## Voice Target

Write English that feels like a strong Latin American bilingual professional: clear, confident, and technically credible. Avoid two extremes:

- Too literal: “I made the realization of dashboards” → unnatural Spanish calque.
- Too native-corporate: “Spearheaded mission-critical enterprise transformation” → sounds inflated if the source was modest.

Use direct, credible phrasing:

- “Built dashboards to monitor sales performance.”
- “Supported data cleaning and reporting workflows.”
- “Developed statistical analyses in R for academic projects.”

## Technical Terminology

Preserve canonical names: R, Python, SQL, LaTeX, Power BI, Excel, Git, TinyTeX, XeLaTeX, Shiny, tidyverse.

Prefer industry equivalents:

| Spanish | Preferred English |
|---|---|
| análisis de datos | data analysis |
| visualización de datos | data visualization |
| modelos estadísticos | statistical models |
| limpieza de datos | data cleaning |
| bases de datos | databases |
| automatización de reportes | report automation |
| indicadores | metrics / indicators, depending on context |
| prácticas profesionales | internship / professional internship |
| proyecto de grado | capstone project / thesis project, depending on context |

## Resume Style

- Start bullets with action verbs: Built, Developed, Analyzed, Automated, Designed, Supported, Implemented.
- Keep bullets one idea each.
- Prefer results and scope over adjectives.
- Keep first person out of bullets unless translating a profile summary that needs it.
- If a metric exists, preserve it exactly. If no metric exists, do not invent one.

## LaTeX Handling

When translating `.tex` CV content:

- Keep commands unchanged: `\cvsection`, `\cventry`, `\begin{cvitems}`.
- Translate only human-readable text inside arguments or item bodies.
- Preserve braces, escaping, accents when needed, and URLs.
- Do not reformat the template unless requested.

## Notes Policy

Add notes only when useful:

- Ambiguous seniority or job title.
- A phrase could be translated two valid ways.
- A claim should be softened or clarified for English-speaking recruiters.
- A literal translation would hurt ATS/searchability.
